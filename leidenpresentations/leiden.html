JGuard Core Concepts

Making Implicit State Explicit

  • Verified Classes: Top-level unit for misuse checks
  • Guards: Boolean variables tracking object state
  • Requirements: Preconditions for method invocation
  • Consequences: State changes after method execution
public verified class Cipher {
    private guard isInitialized;
    private guard encryptionStarted finally false;
    // Guards make object state explicit!
}

Requirements: Preventing Invalid Method Calls

Example: Cipher Initialization

public verified void init(@NonNull Key key)
    require !encryptionStarted, !isInitialized
    when returns sets isInitialized,
    when returns sets encryptionStarted {}

What This Prevents

  • Insecure Parameters: Null key detection
  • Missing Conditions: Re-initialization prevention
  • Incorrect Call Sequence: State validation

How JGuard Works: Generated Checks

JGuard Requirements → Java Runtime Checks

// Generated Java code from JGuard
public void init(@NonNull Key key) {
    if (key == null) reportMisuse();
    if (encryptionStarted) reportMisuse();
    if (isInitialized) reportMisuse();
    // Original method body follows...
}

Runtime checks are automatically inserted at compile time, making APIs defensive by default

Consequences: Managing State Transitions

Example: Iterator Safety

public verified class SafeIterator<T> {
    private guard canCallNext;
    
    public verified boolean hasNext()
        when returns true sets canCallNext {}
        
    public verified T next() require canCallNext
        when returns resets canCallNext {}
}

Prevents calling next() without a valid hasNext() check

The Complete Solution

From Insecure to Secure

Original (Insecure)

Cipher cipher = Cipher.getInstance("AES/ECB/PKCS5Padding");

LLM-Generated Fix (After Static Analysis Feedback)

Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding");
SecretKeySpec keySpec = new SecretKeySpec(key.getEncoded(), "AES");
cipher.init(ENCRYPT_MODE, keySpec, 
    new GCMParameterSpec(128, generateRandomIV()));

Personalized Explanation (Based on Developer Level)

"ECB mode encrypts identical blocks to identical ciphertexts, making patterns visible. GCM mode uses a unique IV for each encryption, ensuring security."
